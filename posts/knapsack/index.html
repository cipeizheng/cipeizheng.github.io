第二部分：4 类模板 + 边界细节详解 #

模板 1️⃣：0-1 背包 · 求最大价值 #

public int knapsack01(int[] weights, int[] values, int capacity) {
    int[] dp = new int[capacity + 1];  // Java 默认初始化为 0
    
    for (int i = 0; i < weights.length; i++) {
        // ⚠️ 关键 1：倒序遍历（防止重复选）
        // ⚠️ 关键 2：从 capacity 到 weights[i]，天然保证 c - weights[i] >= 0
        for (int c = capacity; c >= weights[i]; c--) {
            dp[c] = Math.max(dp[c], dp[c - weights[i]] + values[i]);
            // ✅ 无需 if (c >= weights[i])，循环条件已保证
        }
    }
    return dp[capacity];
}
边界问题解答
为什么 c 从 capacity 开始？0-1 背包需要检查所有容量能否放入当前物品
为什么 c >= weights[i] 就停止？c < weights[i] 时物品放不进，无需更新
需要 if (c >= weights[i]) 吗？❌ 不需要，循环条件已保证
为什么倒序？防止同一物品被重复选取

模板 2️⃣：完全背包 · 求最大价值 #

public int knapsackComplete(int[] weights, int[] values, int capacity) {
    int[] dp = new int[capacity + 1];
    
    for (int i = 0; i < weights.length; i++) {
        // ⚠️ 关键 1：正序遍历（允许重复选）
        // ⚠️ 关键 2：从 weights[i] 开始，天然保证 c - weights[i] >= 0
        for (int c = weights[i]; c <= capacity; c++) {
            dp[c] = Math.max(dp[c], dp[c - weights[i]] + values[i]);
            // ✅ 无需 if (c >= weights[i])，循环起点已保证
        }
    }
    return dp[capacity];
}
边界问题解答
为什么 c 从 weights[i] 开始？c < weights[i] 时物品放不进，跳过无效遍历
需要 if (c >= weights[i]) 吗？❌ 不需要，循环起点已保证
为什么正序？允许同一物品被多次选取

模板 3️⃣：完全背包 · 求方案数 #

3A：组合数（顺序无关） #

public int countCombinations(int[] coins, int amount) {
    int[] dp = new int[amount + 1];
    dp[0] = 1;  // ⚠️ 关键：凑 0 有 1 种方案（什么都不选）
    
    // ⚠️ 关键：物品在外层 → 固定物品顺序 → 组合数
    for (int coin : coins) {
        for (int c = coin; c <= amount; c++) {
            dp[c] += dp[c - coin];
            // ✅ 无需 if (c >= coin)，循环起点已保证
        }
    }
    return dp[amount];
}

3B：排列数（顺序有关） #

public int countPermutations(int[] nums, int target) {
    int[] dp = new int[target + 1];
    dp[0] = 1;
    
    // ⚠️ 关键：容量在外层 → 每个容量自由选择"最后一步" → 排列数
    for (int c = 0; c <= target; c++) {
        for (int num : nums) {
            // ⚠️ 这里需要 if！因为 c 从 0 开始，可能 c < num
            if (c >= num) {
                dp[c] += dp[c - num];
            }
        }
    }
    return dp[target];
}
边界问题组合数 (3A)排列数 (3B)
外层循环物品容量
内层循环起点c = coinc = 0
需要 if (c >= num) 吗？❌ 不需要✅ 需要
为什么？循环起点已保证c 从 0 开始，可能小于 num

模板 4️⃣：完全背包 · 求最小数量 #

public int minCount(int[] items, int capacity) {
    // ⚠️ 关键 1：求最小值，初始化为"不可达"的极大值
    // ⚠️ 关键 2：用 capacity+1 代替 Integer.MAX_VALUE，避免溢出
    int[] dp = new int[capacity + 1];
    Arrays.fill(dp, capacity + 1);
    dp[0] = 0;  // ⚠️ 关键 3：凑 0 需要 0 个物品
    
    for (int item : items) {
        for (int c = item; c <= capacity; c++) {
            dp[c] = Math.min(dp[c], dp[c - item] + 1);
            // ✅ 无需 if (c >= item)，循环起点已保证
            // ✅ 无需 if (dp[c-item] != INF)，capacity+1 不会溢出
        }
    }
    
    // ⚠️ 关键 4：判断是否可达
    return (dp[capacity] > capacity) ? -1 : dp[capacity];
}
边界问题解答
为什么初始化 capacity + 1？最坏情况用 capacity 个 1 凑出，capacity+1 一定更大
为什么不用 Integer.MAX_VALUE？MAX_VALUE + 1 会溢出变负数
需要 if (dp[c-item] != INF) 吗？❌ 不需要，capacity+1 + 1 不会溢出 int
为什么 dp[0] = 0？凑 0 元需要 0 个硬币（基准状态）
最后为什么判断 > capacity？如果仍是初始值，说明无法凑出

🗂️ 第三部分：边界判断速查表（核心！） #

问题 1：什么时候需要 if (c >= weight) 判断？ #

内层循环起点是否需要 if原因
c = 0✅ 需要c 可能小于 weight，下标会负
c = weight❌ 不需要起点已保证 c >= weight
c = weight 到 capacity❌ 不需要全程合法

示例对比：

// ❌ 需要 if（c 从 0 开始）
for (int c = 0; c <= capacity; c++) {
    if (c >= weight) {  // ⚠️ 必须加
        dp[c] = Math.min(dp[c], dp[c - weight] + 1);
    }
}

// ✅ 不需要 if（c 从 weight 开始）
for (int c = weight; c <= capacity; c++) {
    dp[c] = Math.min(dp[c], dp[c - weight] + 1);  // 天然合法
}

问题 2：什么时候初始化 0，什么时候初始化 INF？ #

问题类型初始化策略原因
求最大值0合法方案一定 ≥ 0，能"覆盖"初始值
求最小值INF（capacity+1）合法方案一定 < INF，能"覆盖"初始值
求方案数0，但 dp[0] = 1累加计数，初始为 0；dp[0]=1 是基准

问题 3：什么时候外层遍历物品，什么时候外层遍历容量？ #

目标外层循环内层循环原因
组合数（顺序无关）📦 物品🎒 容量固定物品顺序，避免重复计数
排列数（顺序有关）🎒 容量📦 物品每个容量自由选择"最后一步"
最值问题（min/max）任意（推荐物品）对应正/倒序交换律成立，顺序不影响结果

问题 4：什么时候正序，什么时候倒序？ #

背包类型物品限制遍历方向原因
0-1 背包每件 1 次🔽 倒序 capacity → weight防止重复选
完全背包无限次🔼 正序 weight → capacity允许重复选

🧪 第四部分：完整实战示例 #

LeetCode 322. 零钱兑换（最小值） #

class Solution {
    public int coinChange(int[] coins, int amount) {
        // 1. 初始化：求最小值 → 用 amount+1 代替 MAX_VALUE
        int[] dp = new int[amount + 1];
        Arrays.fill(dp, amount + 1);
        dp[0] = 0;
        
        // 2. 完全背包：物品在外，容量正序
        for (int coin : coins) {
            for (int c = coin; c <= amount; c++) {
                // ✅ 无需 if (c >= coin)，循环起点已保证
                // ✅ 无需 if (dp[c-coin] != INF)，amount+1 不会溢出
                dp[c] = Math.min(dp[c], dp[c - coin] + 1);
            }
        }
        
        // 3. 判断是否可达
        return dp[amount] > amount ? -1 : dp[amount];
    }
}

LeetCode 377. 组合总和Ⅳ（排列数） #

class Solution {
    public int combinationSum4(int[] nums, int target) {
        int[] dp = new int[target + 1];
        dp[0] = 1;
        
        // ⚠️ 排列数：容量在外层
        for (int c = 0; c <= target; c++) {
            for (int num : nums) {
                // ⚠️ 这里需要 if！因为 c 从 0 开始
                if (c >= num) {
                    dp[c] += dp[c - num];
                }
            }
        }
        return dp[target];
    }
}

LeetCode 279. 完全平方数（最小值） #

class Solution {
    public int numSquares(int n) {
        int[] dp = new int[n + 1];
        Arrays.fill(dp, n + 1);
        dp[0] = 0;
        
        // 枚举所有完全平方数作为物品
        for (int x = 1; x * x <= n; x++) {
            int square = x * x;
            // ✅ 从 square 开始，无需 if (c >= square)
            for (int c = square; c <= n; c++) {
                dp[c] = Math.min(dp[c], dp[c - square] + 1);
            }
        }
        return dp[n];
    }
}

💡 终极检查清单（写代码前对照） #

  1. 问题类型？

    • 最大值 → 初始化 0
    • 最小值 → 初始化 capacity+1
    • 方案数 → 初始化 0，dp[0]=1

  2. 物品能重复选吗？

    • 否 → 0-1 背包 → 容量倒序
    • 是 → 完全背包 → 容量正序

  3. 顺序影响结果吗？

    • 组合数 → 物品在外层
    • 排列数 → 容量在外层
    • 最值 → 任意（推荐物品在外）

  4. 循环起点设对了吗？

    • 从 weight 开始 → 无需 if 判断
    • 从 0 开始 → 需要 if (c >= weight)

  5. 结果需要判断可达性吗？

    • 最小值问题 → 检查是否仍为初始值
    • 最大值/计数 → 直接返回 dp[capacity]
