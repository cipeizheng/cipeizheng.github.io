3440. 重新安排会议得到最多空余时间 II #

Rearrange Meetings to Get Maximum Free Time II。这个题目要求我们重新安排会议，以便获得最多的空余时间。它的核心思想是通过两次遍历来计算每个会议的空余时间。

两次遍历的必要性 #

在阅读官方题解后，我一直有一段代码无法理解：

for (int i = 0, t1 = 0, t2 = 0; i < n; i++) {
    if (endTime[i] - startTime[i] <= t1) {
        q[i] = true;
    }
    t1 = Math.max(t1, startTime[i] - (i == 0 ? 0 : endTime[i - 1]));

    if (endTime[n - i - 1] - startTime[n - i - 1] <= t2) {
        q[n - i - 1] = true;
    }
    t2 = Math.max(t2, (i == 0 ? eventTime : startTime[n - i]) - endTime[n - i - 1]);
}

根据题解描述：

q[i] 记录会议 i 是否适用于第一种情况，首先从左到右遍历会议，同时记录当前遍历到的会议 i 左侧非相邻的空余时间段的最大时长 t1，如果 t1 ≥ endTime[i] - startTime[i]，那么说明会议 i 左侧有满足第一种情况的空余时间段，记 q[i] = true。同样地，我们从右到左遍历会议，记下会议 i 右侧是否有满足第一种情况的空余时间段。

这里的第一种情况指的是有一个空隙能够容纳当前会议。但是为什么需要遍历两次？第一次从左到右，第二次从右到左？我认为只需要一次遍历就可以了。因为 i 的取值是从 0 到 n-1，而 n-i-1 的取值是从 n-1 到 0，这两次遍历实际上是对同一组会议进行处理，相当于是把所有的空余时间计算了两遍。

然而我的理解是错误的。实际上，这两次遍历是必要的。因为当i遍历到某个会议时，t1 只记录到了当前会议左侧的空余时间，max函数也只考虑了当前会议左侧的空余时间，而遍历到n-i-1时，t2 只记录到了当前会议右侧的空余时间。在单方向遍历的时候，例如从左到右遍历时，无法得知当前会议右侧的空余时间能不能放得下当前会议。

所以在Leetcode的评论区，有人说这个思路和接雨水的思路类似。

接雨水的思路也是从左到右和从右到左各遍历一次，分别计算左侧和右侧的最大高度，然后再计算每个位置的水量。

同相似问题的对比思考 #

在 LeetCode 上有一个名字类似的题目，叫做 Rearrange Meetings to Get Maximum Free Time I。这个题目和我们现在讨论的题目看起来非常相似，然而它是用滑动窗口的方式来解决的。

为什么看似相同的两个题目采取的思路不同，其实是因为他们的问题约束有一个很大的区别：会议的顺序能不能改变。

在 I 中，会议的顺序需要保持不变，我们可以重新安排至多 k 个会议，这是要保持会议的顺序不变，合并连续的空隙。

而在 II 中，会议的顺序可以改变，我们只能重新安排 1 个会议， 也就是说我们可以将任意的一个会议“拿走”，放到别的空隙里面，这样就“打通”了它原本所在的左右两个空隙。

为什么问题 II 用“两次遍历”？ #

目标：最大化某一段连续空闲时间

  • 我们可以移动一个会议，只要它的长度 ≤ 某个已有空隙，就可以把它“塞”进去。
  • 移动后，它原来占据的位置就空出来了，它左边和右边的空隙就可以连起来，再加上这个会议的时长，形成更大的空隙。

关键判断：这个会议能不能被“移走”？

  • 要判断一个会议能不能被移走，就要看是否存在一个足够大的空隙来容纳它。
  • 所以我们需要：
    • 正向遍历：计算每个位置左边所有空隙的最大值 → 能否移到左边？
    • 反向遍历：计算每个位置右边所有空隙的最大值 → 能否移到右边？
  • 如果会议长度 ≤ 左边或右边的最大空隙，说明它可以被“移走”，那么它原来的位置就可以被释放。

为什么是两次遍历？这个问题我们在前面已经讨论过了。

为什么问题 I 用“滑动窗口”？ #

目标：合并连续的 k+1 个间隙

  • 不能改变顺序，所以会议之间的相对位置不能乱。
  • 但我们可以“移走”最多 k 个会议，它们原来占据的位置就空出来了。
  • 由于顺序不能变，我们只能把会议“腾空”，但不能打乱顺序。
  • 所以，如果我们移走 k 个会议，最多可以把 k+1 个连续的间隙连起来。

所以问题变成了：

在 n+1 个间隙中，选出连续的 k+1 个，使得它们的和最大。

这就是一个经典的 滑动窗口（Sliding Window） 问题。

总的来说，还是要根据问题的约束条件来选择合适的算法思路，虽然两个题目名字上开起来有递进关系，但实际上它们的解法和思路是完全不同的。
