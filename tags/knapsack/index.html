背包问题: 模板 + 边界问题

第一部分：核心变量定义 变量 含义 Java 类型 capacity 背包容量 / 目标值（amount、target、n） int weights[] / coins[] 物品重量 / 硬币面额 int[] dp[] 核心状态：dp[x] = 容量为 x 时的最优解 int[] 第二部分：4 类模板 + 边界细节详解 模板 1️⃣：0-1 背包 · 求最大价值 public int knapsack01(int[] weights, int[] values, int capacity) { int[] dp = new int[capacity + 1]; // Java 默认初始化为 0 for (int i = 0; i < weights.length; i++) { // ⚠️ 关键 1：倒序遍历（防止重复选） // ⚠️ 关键 2：从 capacity 到 weights[i]，天然保证 c - weights[i] >= 0 for (int c = capacity; c >= weights[i]; c--) { dp[c] = Math.max(dp[c], dp[c - weights[i]] + values[i]); // ✅ 无需 if (c >= weights[i])，循环条件已保证 } } return dp[capacity]; } 边界问题 解答 为什么 c 从 capacity 开始？ 0-1 背包需要检查所有容量能否放入当前物品 为什么 c >= weights[i] 就停止？ c < weights[i] 时物品放不进，无需更新 需要 if (c >= weights[i]) 吗？ ❌ 不需要，循环条件已保证 为什么倒序？ 防止同一物品被重复选取 模板 2️⃣：完全背包 · 求最大价值 public int knapsackComplete(int[] weights, int[] values, int capacity) { int[] dp = new int[capacity + 1]; for (int i = 0; i < weights.length; i++) { // ⚠️ 关键 1：正序遍历（允许重复选） // ⚠️ 关键 2：从 weights[i] 开始，天然保证 c - weights[i] >= 0 for (int c = weights[i]; c <= capacity; c++) { dp[c] = Math.max(dp[c], dp[c - weights[i]] + values[i]); // ✅ 无需 if (c >= weights[i])，循环起点已保证 } } return dp[capacity]; } 边界问题 解答 为什么 c 从 weights[i] 开始？ c < weights[i] 时物品放不进，跳过无效遍历 需要 if (c >= weights[i]) 吗？ ❌ 不需要，循环起点已保证 为什么正序？ 允许同一物品被多次选取 模板 3️⃣：完全背包 · 求方案数 3A：组合数（顺序无关） public int countCombinations(int[] coins, int amount) { int[] dp = new int[amount + 1]; dp[0] = 1; // ⚠️ 关键：凑 0 有 1 种方案（什么都不选） // ⚠️ 关键：物品在外层 → 固定物品顺序 → 组合数 for (int coin : coins) { for (int c = coin; c <= amount; c++) { dp[c] += dp[c - coin]; // ✅ 无需 if (c >= coin)，循环起点已保证 } } return dp[amount]; } 3B：排列数（顺序有关） public int countPermutations(int[] nums, int target) { int[] dp = new int[target + 1]; dp[0] = 1; // ⚠️ 关键：容量在外层 → 每个容量自由选择"最后一步" → 排列数 for (int c = 0; c <= target; c++) { for (int num : nums) { // ⚠️ 这里需要 if！因为 c 从 0 开始，可能 c < num if (c >= num) { dp[c] += dp[c - num]; } } } return dp[target]; } 边界问题 组合数 (3A) 排列数 (3B) 外层循环 物品 容量 内层循环起点 c = coin c = 0 需要 if (c >= num) 吗？ ❌ 不需要 ✅ 需要 为什么？ 循环起点已保证 c 从 0 开始，可能小于 num 模板 4️⃣：完全背包 · 求最小数量 public int minCount(int[] items, int capacity) { // ⚠️ 关键 1：求最小值，初始化为"不可达"的极大值 // ⚠️ 关键 2：用 capacity+1 代替 Integer.MAX_VALUE，避免溢出 int[] dp = new int[capacity + 1]; Arrays.fill(dp, capacity + 1); dp[0] = 0; // ⚠️ 关键 3：凑 0 需要 0 个物品 for (int item : items) { for (int c = item; c <= capacity; c++) { dp[c] = Math.min(dp[c], dp[c - item] + 1); // ✅ 无需 if (c >= item)，循环起点已保证 // ✅ 无需 if (dp[c-item] != INF)，capacity+1 不会溢出 } } // ⚠️ 关键 4：判断是否可达 return (dp[capacity] > capacity) ? -1 : dp[capacity]; } 边界问题 解答 为什么初始化 capacity + 1？ 最坏情况用 capacity 个 1 凑出，capacity+1 一定更大 为什么不用 Integer.MAX_VALUE？ MAX_VALUE + 1 会溢出变负数 需要 if (dp[c-item] != INF) 吗？ ❌ 不需要，capacity+1 + 1 不会溢出 int 为什么 dp[0] = 0？ 凑 0 元需要 0 个硬币（基准状态） 最后为什么判断 > capacity？ 如果仍是初始值，说明无法凑出 🗂️ 第三部分：边界判断速查表（核心！） 问题 1：什么时候需要 if (c >= weight) 判断？ 内层循环起点 是否需要 if 原因 c = 0 ✅ 需要 c 可能小于 weight，下标会负 c = weight ❌ 不需要 起点已保证 c >= weight c = weight 到 capacity ❌ 不需要 全程合法 示例对比： ...
