Java 泛型的通配符与PECS原则

Java泛型中的extends与super Java在泛型中引入extends和super通配符，其核心目的是为了在保证类型安全的前提下，增加泛型代码的灵活性，解决泛型类型固有的“不可变性”（Invariance）所带来的限制。而**PECS原则（Producer Extends, Consumer Super）**则是对如何正确使用这两种通配符的高度概括和实用总结。 在Java中，泛型类型默认是不可变的。这意味着，即使String是Object的子类，List<String>也不是List<Object>的子类。 让我们来看一个的例子： // 假设我们有一个打印Object列表的方法 public void printList(List<Object> list) { for (Object obj : list) { System.out.println(obj); } } // 现在我们有一个String列表 List<String> strList = new ArrayList<>(); strList.add("Hello"); strList.add("World"); // 下面的调用将会导致编译错误！ // printList(strList); // 编译不通过 这个编译错误可能会让初学者感到困惑。编译器之所以阻止这种行为，是为了防止潜在的类型安全问题。如果printList(strList)是合法的，那么在printList方法内部，我们就可以向这个列表中添加任何Object类型的对象，例如一个Integer： public void printList(List<Object> list) { list.add(123); // 合法，因为123是Object的子类 for (Object obj : list) { System.out.println(obj); } } 这样一来，当我们回到调用方，尝试从原来的strList中取出一个元素时，就可能会遇到ClassCastException，因为我们期望得到一个String，但实际上取出的可能是一个Integer。例如下面的代码： List<String> strList = new ArrayList<>(); strList.add("Hello"); strList.add("World"); printList(strList); // 假设这行代码编译通过了，那么strList的内容是["Hello", "World", 123] String s = strList.get(3); // 这里会抛出ClassCastException 于是，Java的泛型设计在编译期就杜绝了这种风险，禁止将List<String>传递给List<Object>类型的方法。 ...
