剖析跳跃游戏 I && II

跳跃游戏 I 和 II 在 LeetCode 上有两个跳跃游戏的题目，分别是 Jump Game I 和 Jump Game II。这两个题目都涉及到如何在一个数组中跳跃到最后一个元素，而且都用到了贪心算法的思路。这两个问题的描述几乎如出一辙：给定一个非负整数数组，每个元素代表你能跳跃的最大长度，判断你是否能从起点到达终点。然而，尽管它们的表面描述相似，但它们的解法却截然不同。 跳跃游戏 I 问的是：你是否能到达最后一个下标？ 跳跃游戏 II 问的是：到达最后一个下标的最小跳跃次数是多少？ 跳跃游戏 I (canJump)：探索可能性的边界 我们先来看问题相对简单的一方：跳跃游戏 I。它的目标是回答一个“是”或“否”的问题。 问题描述 (#55. Can Jump) 给你一个非负整数数组 nums，你最初位于数组的第一个下标。数组中的每个元素代表你在该位置可以跳跃的最大长度。判断你是否能够到达最后一个下标，如果可以，返回 true；否则，返回 false。 要解决这个问题，我们只需要找到任何一条可以到达终点的路径即可。这引导我们采用一种目光长远的贪心策略：在任何时刻，我只关心我能到达的最远“势力范围”有多大。 下面是一个非常高效且优雅的贪心解法： class Solution { public boolean canJump(int[] nums) { // cover 代表从起点开始，我们能够到达的最远位置的索引 int cover = 0; if (nums.length == 1) { return true; } // 循环条件是关键: i <= cover // 我们的“脚步” i，必须在我们已知的、能够到达的范围之内 for (int i = 0; i <= cover; i++) { // 在当前位置 i，尝试更新最远覆盖范围 cover = Math.max(cover, i + nums[i]); // 如果覆盖范围已经到达或超过了终点，任务完成 if (cover >= nums.length - 1) { return true; } } // 如果循环正常结束，说明我们的脚步 i 跑出了 cover 的范围 // 意味着前方出现了无法跨越的“鸿沟”，无法到达终点 return false; } } 这个算法的精髓在于 cover 变量和 for (int i = 0; i <= cover; i++) 这个循环条件。 ...
